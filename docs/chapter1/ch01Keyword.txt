- TCP
    
    웹에서 주로 사용하는 reliable하고 안정적인 방식의 전송계층 프로토콜
    
- UDP
    
    빠르지만 파일의 손실 또는 순서가 맞지 않을 가능성이 있는 불안정한 방식의 전송계층 프로토콜
    
- 시스템 콜?
    
    하드웨어 명령어를 사용하기 위해 커널모드에 접근할 수 있도록 권한을 요청하는 것
    
- 하드웨어 인터럽트
    
    하드웨어에서 발생시키는 인터럽트, 입출력 장치가 CPU에 어떠한 사실을 알리거나 작업 완료를 알리기 위해 발생시킴
    
- 리눅스의 파일과 파일 디스크립터
    
    리눅스는 모든 것을 파일로 처리하며 파일 디스크럽터는 파일을 표현하는 고유 번호
    
- socket() 시스템 콜
    
    소켓의 IP 도메인, 타입, 프로토콜을 설정해주고 파일 디스크럽터를 리턴 받음
    
- bind() 시스템 콜
    
    생성된 소켓에 포트번호와 IP주소를 부여하는 것(클라이언트는 생성시 자동으로 포트번호가 부여됨)
    
- listen() 시스템 콜
    
    TCP에서만 사용하는 것으로 연결요청을 받아주기 위한 backlog queue를 설정해주는 것
    
- accept() 시스템 콜
    
    listen에서 받은 syn 요청에 대한 ACK메세지 응답을 통해 3-way-handshake를 완성해주는 것
    
- 멀티 프로세스
    
    프로그램의 성능을 위해 동시에 처리(예를 들면, 송신과 수신)할 수 있도록 프로세스를 여러개 돌리는 것
    
- 병렬 처리
    
    멀티 프로세스를 통해 여러가지 과업을 동시에 처리하는 것
